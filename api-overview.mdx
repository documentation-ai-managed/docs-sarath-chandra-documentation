## Authentication

<Callout kind="alert">All API requests require authentication using an API key for security.</Callout>

<Tabs>
  <Tab title="API Key Setup" icon="key">
    <Steps>
      <Step title="Generate Key" icon="settings">
        Access your account settings and generate a new API key.
      </Step>
      <Step title="Include in Requests" icon="send">
        Add the key to the Authorization header for all requests.
      </Step>
      <Step title="Secure Storage" icon="lock">
        Store API keys securely and rotate them regularly.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Request Format" icon="code">
    <CodeGroup tabs="cURL,JavaScript">
      ````bash
      curl -H "Authorization: Bearer YOUR_API_KEY" \
           -H "Content-Type: application/json" \
           https://api.sarathchandra.com/v1/projects
      ````
      ````javascript
      const response = await fetch('https://api.sarathchandra.com/v1/projects', {
        headers: {
          'Authorization': 'Bearer YOUR_API_KEY',
          'Content-Type': 'application/json'
        }
      });
      
      const projects = await response.json();
      ````
    </CodeGroup>
  </Tab>
</Tabs>

## Core Endpoints

The API provides endpoints for managing projects, pages, and user permissions. All endpoints support standard HTTP methods and return consistent JSON responses.

<ExpandableGroup>
  <Expandable title="Projects Endpoints" default-open="true">
    <p>Create, update, and manage documentation projects.</p>
    <ul>
      <li>GET /projects - List all projects</li>
      <li>POST /projects - Create new project</li>
      <li>GET /projects/{id} - Get project details</li>
      <li>PUT /projects/{id} - Update project</li>
      <li>DELETE /projects/{id} - Delete project</li>
    </ul>
  </Expandable>
  <Expandable title="Pages Endpoints" default-open="false">
    <p>Manage individual documentation pages within projects.</p>
    <ul>
      <li>GET /projects/{id}/pages - List project pages</li>
      <li>POST /projects/{id}/pages - Create new page</li>
      <li>GET /pages/{id} - Get page content</li>
      <li>PUT /pages/{id} - Update page</li>
      <li>DELETE /pages/{id} - Delete page</li>
    </ul>
  </Expandable>
  <Expandable title="Users Endpoints" default-open="false">
    <p>Manage user accounts and permissions.</p>
    <ul>
      <li>GET /users - List users</li>
      <li>POST /users - Create user</li>
      <li>GET /users/{id} - Get user details</li>
      <li>PUT /users/{id} - Update user</li>
    </ul>
  </Expandable>
</ExpandableGroup>

## Error Handling

The API returns standard HTTP status codes and detailed error messages in JSON format. Handle errors gracefully in your applications.

<CodeGroup tabs="JavaScript,Python">
  ````javascript
  try {
    const response = await fetch('/api/projects');
    if (!response.ok) {
      const error = await response.json();
      console.error('API Error:', error.message);
    } else {
      const data = await response.json();
      console.log('Success:', data);
    }
  } catch (err) {
    console.error('Network Error:', err);
  }
  ````
  ````python
  import requests
  
  try:
      response = requests.get('/api/projects')
      response.raise_for_status()
      data = response.json()
      print('Success:', data)
  except requests.exceptions.RequestException as err:
      print('Error:', err)
  ````
</CodeGroup>

<Expandable title="Common Error Codes" default-open="false">
  - 400 Bad Request: Invalid request parameters
  - 401 Unauthorized: Missing or invalid API key
  - 403 Forbidden: Insufficient permissions
  - 404 Not Found: Resource does not exist
  - 429 Too Many Requests: Rate limit exceeded
  - 500 Internal Server Error: Server-side issue
</Expandable>
